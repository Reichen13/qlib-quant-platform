# 审查基准尺(阶段2 · 第一性原理拆解)

> 本文件在打开任何源码之前写下,作为后续对照代码的"应该是什么"基准。
> 写作日期:2026-07-04。审查对象:Qlib Quant Platform(用户自建盘后量化分析系统)。

## 0. 用户画像(阶段1确认结果)

- 资金量:10~50 万人民币。含义:流动性基本不是约束(日成交 1 亿以上的票,单笔 5 万冲击可忽略);但**一手门槛真实存在**(百元股一手 1 万+,50 万资金买 10 只百元股每只只能 3~4 手,离散化误差大);**最低佣金 5 元**意味着单笔低于 2 万的交易实际费率高于万 2.5。
- 持仓周期:波段,几天~几周。含义:红利税落在 20%/10% 惩罚区;因子半衰期必须匹配 5~20 个交易日;隔夜跳空风险无法对冲(无两融)。
- 回撤承受:约 20%(行为上)。含义:系统应在组合回撤接近 10~15% 时就给降仓提示,因为行为割肉点通常先于口头承受度到达。
- 费用:佣金万 2.5 + 最低 5 元,印花税卖出 0.05%,过户费双向 0.001%。单次往返摩擦约 0.11%~0.13%(不含滑点)。**周度全换仓策略年摩擦 6%+,必须显式计入。**
- 无两融:不能做空、不能加杠杆。个人优势 = 可空仓、灵活、无合规约束。系统应放大"该不该空仓"的判断能力,而不是堆多空因子。
- 用户自认四大短板:数据准确性、回测/信号可信度、输出太多难执行、持仓管理缺失。**四项全中,说明系统目前更像"研究玩具"而非"实战工具",审查须以此为默认怀疑方向。**

## 1. 这套系统本质上应该回答的五个问题(合格线)

1. **今晚复盘,我该看哪 1~5 只票?** —— 不是全市场扫描结果列表。
2. **我手里的持仓,明天开盘该卖/减/加/持有?** —— 持仓管理优先于选股。系统必须知道我的持仓。
3. **当前市场环境适合我这种波段风格吗?该放大还是收缩敞口?** —— regime 判断 + 仓位建议。
4. **我的策略最近是真失效还是正常回撤?** —— 信号胜率/IC 的滚动监控 + 熔断建议。
5. **系统给的每个信号,能用一句人话解释吗?** —— 解释不了 = 拿不住 = 高买低卖。

任何一条回答不了,即为一级缺陷,无论因子多 fancy。

## 2. 四层理想形态(对照尺)

### 2.1 数据层理想形态
- 每根 K 线可回答:什么复权方式、何时复权、从哪个原始字段来。前复权数据每次除权后全历史漂移,缓存的技术指标必须同步失效。
- 样本带生命周期:上市日(过滤次新)、ST 区间、停牌区间、退市(survivorship bias)、分红送配。
- 更新幂等:跑两次 = 跑一次。缺失显式标记为 NaN/null,绝不默认 0 或前值填充后不留痕。
- 停牌日不产生假行情;复牌首日涨跌幅不当成正常交易日信号。
- 财务数据用公告日,不用报告期(否则 look-ahead)。

### 2.2 因子层理想形态
- 每个因子有身份证:经济含义、公式、窗口、预期方向、半衰期、适用环境。
- 半衰期必须匹配用户 5~20 日持仓周期;衰减快于 2 日的因子对该用户无用。
- 横截面可比:行业/市值中性化(A 股小盘效应极强,不做市值中性 = 因子暴露实为规模暴露)。
- 无未来函数:标准化参数只用样本内;停牌期间滚动窗口处理显式。
- 已知共线簇,不假装多元。

### 2.3 回测层理想形态
- 撮合保守显式:T+1;涨停买不到、跌停卖不掉、一字板双向不可成交;停牌跳过;成交价口径(开/收/VWAP)写清;信号日次日成交。
- 成本 = 万 2.5 双向 + 5 元最低 + 印花税 0.05% 卖出 + 过户费,按历史真实费率;滑点至少固定 0.1%+,小盘更高。
- 评估三问:赚不赚(净值)、为何赚(归因到行业/风格/择时)、可持续吗(样本外/walk-forward/参数±20%稳定性)。
- 回测 ≠ 策略:没有独立经济学逻辑的漂亮回测标记为高风险。

### 2.4 工程层理想形态
- 失败可 5 分钟定位:数据问题/代码问题/市场问题。
- 可复现:同一输入+同一 commit → 同一报告;随机过程有种子;数据版本绑定回测结果。
- 可降级:单一数据源(baostock/akshare/yfinance/LLM)挂了,基础复盘流程仍能出报告。
- 关键中间产物(因子值表、信号表)落盘可抽查。

## 3. 预登记的"待验证假设"(读码前的先验怀疑)

基于 README/CLAUDE.md 已声明的架构(不算读源码),先登记以下怀疑点,阶段 3 逐一取证:

- H1:Qlib bin 数据由 baostock 增量写入,**复权口径是否一致**(baostock 默认前复权?每次除权后历史是否重写?)——若增量追加而不重写历史,前复权序列会出现拼接断层,直接污染所有技术指标与回测。
- H2:回测用 Alpha158+LightGBM 走 Qlib 默认 executor,**T+1/涨跌停/停牌约束大概率未启用**(Qlib 默认配置不含 A 股涨跌停撮合)。
- H3:横截面处理(CSRankNorm 等)fit 区间可能覆盖测试集 → 未来函数。
- H4:全市场覆盖与 CSI300 口径混用已被 CLAUDE.md 自己承认为风险 → 股票池/因子样本可能仍有 survivorship/样本选择偏差(退市股、ST、次新是否过滤未知)。
- H5:系统有大量"广度型"功能(热点、配对、ETF、辩论、新闻),但持仓管理、健康度监控、注意力聚焦大概率缺失(用户四短板自证)。
- H6:成本模型可能是固定万 3 单边 + 无最低 5 元、无印花税区分买卖。
- H7:多数据源 fallback(baostock/akshare/yfinance)拼接的历史一致性未验证。
- H8:LLM 类功能(智能体辩论、AI 策略)对 P&L 的边际价值可能为负(消耗注意力+过度自信),属"洁癖项"候选而非改进项。

## 4. 杀伤力排序原则(本次审查专用)

对该用户,缺陷杀伤力从高到低的先验排序:
1. 让回测虚高、导致实盘按假信号下单的缺陷(未来函数、复权断层、涨跌停可成交、成本缺失)。
2. 持仓管理缺失(用户拿着票不知道何时卖 —— 波段交易者的主要亏损来源是出场,不是进场)。
3. 输出过载(注意力被 50 只候选稀释 → 漏单、追高、计划外交易)。
4. 失效监控缺失(策略死了还在用)。
5. 数据覆盖/口径混乱(影响研究质量但不直接下单)。
6. 工程洁癖(除非影响上述五项)。
