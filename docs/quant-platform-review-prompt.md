# 量化分析平台全面审查 Prompt

你是一位在顶级量化对冲基金（如 Citadel/Two Sigma/WorldQuant）有 10 年以上经验的量化研究总监。请以专业量化专家的视角，对以下量化分析平台进行全面、深入的审视。

---

## 平台概览

这是一个基于 Qlib + FastAPI + React 构建的 A 股量化分析平台，目前部署在云端服务器，数据源为 Qlib cn_data（CSI300 成分股，Alpha158 因子体系）+ akshare（宏观数据）+ Baostock（行业分类）+ yfinance（部分美股）。

---

## 审查要求

请逐一审视以下每个维度，对每个功能给出 **"保留/改进/移除/新增"** 的判断及详细理由：

### 一、因子分析模块 (Factor Analysis)

**已有功能：**
1. Alpha158 因子 IC 分析（Spearman Rank IC，可选行业中性化）
2. 增强统计面板：偏度、峰度、t 检验、信息比率、IC 自相关
3. 行业加权 IC 分解
4. IC 衰减分析（1/3/5/10/20 日多周期）
5. 因子相关性分析（IC 序列相关系数矩阵）
6. IC 加权信号组合评分（Top 10 选股）

**请审视：**
- 整个因子研究链条是否完整？缺少什么关键步骤？
- 行业中性化目前只有 OLS，是否需要市值中性化？其他中性化方法是否有必要？
- IC 分析之外，是否应该加入收益率分层分析（quantile return / long-short return）？
- 因子间相关性分析和 IC 衰减分析在实际投资决策中如何使用？当前实现是否有遗漏？
- Alpha158 虽然覆盖面广，但其中大量因子高度冗余。是否需要因子筛选/降维机制？
- 训练集和测试集划分是否合理？是否存在前视偏差（look-ahead bias）？
- 现有实现中哪些功能是"看起来有用但实际决策中用不上"的？

### 二、回测模块 (Backtesting)

**已有功能：**
1. LightGBM/XGBoost 模型训练
2. TopK 选股策略回测
3. 净值曲线、回撤曲线
4. 收益指标：总收益、年化收益、夏普比率、Calmar 比率、最大回撤
5. 统计检验：t 统计量、p 值、信息比率、Sortino 比率、月度胜率
6. 交易成本：佣金 + 滑点
7. 风控参数：单票仓位上限、止损
8. 股票推荐（买卖建议 + 仓位建议）

**请审视：**
- 回测框架中是否存在幸存者偏差（survivorship bias）？
- 当前的回测是截面选股还是时间序列？是否存在未来函数泄露？
- 交易成本模型是否足够（仅固定佣金+滑点 vs 实际市场冲击成本）？
- 是否缺少基准对比（如沪深 300 指数）？超额收益归因是否完整？
- 止损机制是日终检查还是日内检查？实际可执行性如何？
- 回测结果的统计显著性检验是否足够？是否需要样本外测试或交叉验证？
- 对于 A 股市场特有的停牌、涨跌停板限制，回测是否考虑了这些约束？
- 是否需要 walk-forward 回测或多时段交叉验证？

### 三、组合优化模块 (Portfolio Optimization)

**已有功能：**
1. 均值-方差优化（最大夏普比率）
2. 最小方差组合
3. 风险平价
4. 等权配置（基准）
5. 有效前沿可视化
6. 单票权重上限约束

**请审视：**
- 预期收益和协方差矩阵是如何估计的？样本协方差还是 shrinkage 估计？
- 是否有交易成本/换手率约束？优化结果是否会导致不切实际的高换手？
- Black-Litterman 模型是否比纯均值-方差更适合实际应用？
- 是否需要加入更多约束（行业集中度、流动性、最小权重）？
- 风险平价的协方差估计是否稳健？是否有风险预算（risk budgeting）的扩展？
- 有效前沿展示是否代表真实的投资可行集？

### 四、风险管理模块 (Risk Management)

**已有功能：**
1. VaR（95%/99%）和 CVaR
2. 压力测试（6 种历史情景）
3. 相关性矩阵
4. 凯利公式头寸规模
5. 波动率锥
6. 盈亏比和胜率

**请审视：**
- VaR/CVaR 的计算方法是什么？（历史模拟/参数法/蒙特卡洛）参数是否合适？
- 压力测试情景是否覆盖 A 股市场的主要风险事件（2015 股灾、2018 去杠杆、2020 疫情、2024 量化危机）？
- 相关性矩阵在危机时期会急剧变化（correlation breakdown），当前实现是否只依赖历史平均？
- 凯利公式在实际中往往高估仓位，是否有分数凯利或考虑参数不确定性的调整？
- 是否缺少流动性风险、集中度风险、杠杆风险的评估？
- 波动率锥的实际用途是什么？在当前平台中是否只是一个"摆设"？

### 五、宏观模块 (Macro Dashboard)

**已有功能：**
1. 8 个宏观指标（标普 500、纳斯达克、SP500 波动率替代 VIX、美债 10Y、美元指数、黄金、原油、道琼斯）
2. 市场状态分类（增长/通胀双维度四象限）
3. 全天候配置建议
4. 历史数据

**请审视：**
- 仅用 8 个美股/全球指标来指导 A 股投资，方法论上是否有根本性问题？
- akshare 能提供大量 A 股宏观指标（CPI/PMI/M2/社融/利率等），为什么这些反而没有被纳入？
- 四象限框架（桥水全天候）是否适合 A 股市场？A 股的政策驱动特征是否被忽略？
- 如何从宏观状态映射到具体的仓位调整？这一决策链条是否完整？
- 宏观指标的更新频率和滞后性如何？实际可操作性如何？

### 六、策略模块 (Strategy)

**已有功能：**
1. 均值回归扫描（RSI + 布林带）
2. 配对交易（协整检验 + 价差分析）
3. ETF 轮动（基于动量和信号）
4. ETF 筛选器

**请审视：**
- 这些策略是独立实现的还是可以组合？信号冲突如何处理？
- 均值回归的 RSI/布林带参数是否可以优化？是否有过拟合风险？
- 配对交易的协整关系是静态还是动态？样本外稳定性如何？
- ETF 轮动信号的具体计算方法是什么？回测结果是否有统计意义？
- 是否缺少因子策略（如多因子选股）、事件驱动策略、趋势跟踪策略？
- 这些策略的可执行性如何（A 股 T+1、涨跌停、融券限制）？

### 七、数据架构与工程

**请审视：**
- Qlib cn_data 的数据质量和更新频率如何？是否有数据清洗验证机制？
- 多个数据源（Qlib + akshare + Baostock + yfinance）之间的数据一致性如何保证？
- 因子计算（Alpha158）的性能瓶颈在哪？158 个因子是否全部需要预先计算？
- 前端查询的延迟是否可接受？是否有缓存层？
- 是否有数据异常监控和告警？数据断供时的降级策略是什么？
- 回测结果的存储和复现机制是怎样的？

### 八、投资决策链条完整性

从量化投资的标准工作流审视：

```
数据获取 → 数据清洗 → 特征工程 → 因子研究 → 模型训练 → 
回测验证 → 组合优化 → 风险归因 → 执行监控 → 绩效分析
```

**请指出：**
- 哪些环节已经覆盖？哪些环节缺失？
- 缺失环节中，哪些是"必须有"的，哪些是"锦上添花"的？
- 现有各环节之间的衔接是否流畅？是否存在"断层"？
- 对于一个实盘决策，操作者需要在哪些环节之间手动转换数据或格式？

### 九、总体评价

请给出：
1. **平台定位建议**：这个平台最适合做什么？（研究探索 / 策略回测 / 实盘决策支持 / 投教演示？）
2. **Top 5 最急需改进的问题**（按紧迫性排序）
3. **Top 3 应该移除或降级的功能**（投入产出比低）
4. **如果只做一件事**来最大程度提升平台的专业性和实用性，你会建议什么？
