# 腾讯云服务器访问与授权说明

这份说明用于把量化平台部署到腾讯云服务器时，安全地提供登录信息。不要把服务器密码、私钥内容、API Key 直接写进 GitHub、README、提交记录或公开聊天截图。

## 1. 推荐提供的信息

优先使用 SSH 登录。你只需要提供这些信息：

```text
公网 IP：49.235.215.39
SSH 端口：通常是 22，如你改过请填写实际端口
用户名：例如 ubuntu / root / jason
登录方式：密码 / 私钥 / 腾讯云 WebShell
项目目录：如果知道，填写；不知道也可以不填，我会先只读查找
```

如果使用密码登录，建议在临时会话里提供，不要写进项目文件。部署完成后可以改密码或关闭密码登录。

如果使用私钥登录，不要把私钥文本贴到聊天里。更稳妥的方式是：

1. 把私钥文件保存到本机，例如：

   ```text
   C:\Users\Jason\.ssh\tencent-qlib.pem
   ```

2. 告诉我这个本机路径、服务器 IP、端口和用户名。
3. 我会用该文件登录，不会把私钥写进项目或提交到 GitHub。

## 2. 在腾讯云控制台查看登录信息

1. 打开腾讯云控制台。
2. 进入“云服务器 CVM”。
3. 找到运行量化平台的实例。
4. 记录实例的公网 IP。
5. 点实例详情，查看操作系统用户名。常见情况：

- Ubuntu 镜像通常是 `ubuntu`。
- 部分镜像或手工配置可能使用 `root`。
- 如果你创建过自定义用户，也可能是你设置的用户名。

## 3. 如果你不知道 SSH 密码

可以在腾讯云控制台重置实例密码：

1. 进入 CVM 实例列表。
2. 选择目标实例。
3. 找到“重置密码”。
4. 设置一个临时强密码。
5. 重启实例或按腾讯云提示完成密码生效。

注意：重启会短暂影响服务器上的所有服务。因为服务器还有其他项目，重启前请确认当前没有重要任务运行。

## 4. 使用腾讯云 WebShell

如果你不想提供 SSH 密码或私钥，可以使用腾讯云 WebShell：

1. 在 CVM 实例页面点击“登录”。
2. 选择“标准登录”或“WebShell”。
3. 进入终端后，把我给出的只读检查命令粘贴进去执行。
4. 把输出结果发给我，我再判断下一步。

这种方式最安全，但需要你手动复制命令和结果。

## 5. 我上线时会遵守的边界

上线时只处理量化平台相关范围：

- 项目代码目录，例如 `~/quant-platform` 或服务器实际项目目录。
- 前端静态目录，例如 `/var/www/quant`。
- 量化平台后端容器，例如 `quant-backend`。
- 该项目使用的 Qlib 数据目录，例如 `~/.qlib/qlib_data/cn_data`。

不会主动修改：

- 服务器上其他项目目录。
- 系统级 Nginx 全局配置，除非只做语法检查或你明确授权。
- 其他 Docker 容器。
- 密钥、证书、云账号配置。

## 6. 首次登录后会先做什么

首次登录后先只读确认，不会立即改文件：

```bash
pwd
hostname
docker ps --format 'table {{.Names}}\t{{.Status}}\t{{.Ports}}'
find ~ -maxdepth 4 -type f -name update_cn_data.py 2>/dev/null
find ~ -maxdepth 4 -type f -name docker-compose.yml 2>/dev/null
curl -s http://127.0.0.1:8001/health
curl -s 'http://127.0.0.1:8001/api/quote/600519?frequency=daily&indicators=true' | head -c 500
```

确认项目目录后，再做备份、拉取 GitHub 最新提交、部署、单只股票验证。

## 7. 部署后的第一条验证命令

代码部署完成后，可在项目目录执行：

```bash
bash scripts/verify_current_fixes.sh
```

它会检查：

- 后端健康状态。
- `600519` 是否还有 0 值 OHLC。
- 因子分析提交接口是否快速返回 `task_id`。

如果 `600519` 仍有 0 值，再执行定向修复：

```bash
python update_cn_data.py --code sh600519 --start 2026-03-20 --end 2026-06-19 --rebuild-stale
```

只在这一步确认有效后，才考虑扩大到更多股票。
