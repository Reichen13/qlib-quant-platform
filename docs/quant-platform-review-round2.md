## 二、第一轮遗留 + 第二轮新发现

### 🔴 严重（4项）

#### 1. 回测缺少绩效归因分析
- **来源：** 第一轮 二-4 + 八-断裂带2
- **现状：** 回测结果有净值曲线、benchmark曲线、年化收益、夏普、回撤等指标，但没有显式的超额收益分解。用户看到"赚了15%跑赢基准5%"，但不知道为什么——是行业配置贡献的还是选股贡献的？
- **建议：** 加入简化版 Brinson 归因或因子归因表。至少展示：
  - 超额收益 = 行业配置收益 + 选股收益
  - 每月/每季度的归因分解表
  - 如果数据允许，增加风格因子暴露分析（Barra CNE5 风格因子）
- **理由：** 没有归因就没有改进方向。这是"研究工具"和"决策工具"的分水岭。

#### 2. 压力测试场景库缺乏 A 股系统性风险事件
- **来源：** 第一轮 四-1
- **现状：** 当前5个场景中仅"2015 A股股灾"是A股特定事件。缺少：
  - **2016年1月熔断**（4天2次熔断，上证-25%）
  - **2018年去杠杆熊市**（上证从3587跌至2449，持续10个月）
  - **2024年1-2月量化集中平仓**（微盘流动性危机，DMA策略大规模撤出）
  - **2021年教育双减**（中证教育指数-70%）/ **游戏版号暂停**
  - **中美贸易战升级**（2018-2019分阶段关税冲击）
- **建议：** 建立 `STRESS_SCENARIOS` 配置字典，每个场景附带：
  - 冲击幅度（%）、持续时间（天）、恢复时间（天）
  - 受影响最大的行业和风格因子
  - 实际历史日期区间，方便提取真实收益数据
- **理由：** 压力测试的价值 = 情景的真实性和覆盖度。A股特有的政策风险和流动性危机比全球宏观冲击更致命。

#### 3. 宏观→配置映射无回测验证
- **来源：** 第一轮 五-2
- **现状：** `/api/macro/allocation` 返回的配置权重来自 `CN_ALLOCATION_MAP` 字典——这是桥水全天候框架的教科书映射（复苏=增持股票、过热=增持商品等），没有用A股历史数据做过回测验证。
- **建议：** 
  - 用2016-2025共10年的中国宏观状态标签，回测每种状态下不同配置方案的收益
  - 生成"基于证据的配置建议"（evidence-based），与当前"教科书建议"并列展示
  - 最低限度：在前端标注"此配置基于桥水全天候框架，未经A股历史数据回测验证"
- **理由：** 桥水美国全天候不等于中国全天候。教科书映射在中国可能完全失效。

#### 4. 市值中性化缺失
- **来源：** 第一轮 一-2（当时标记为🟡，升为🔴）
- **现状：** 行业中性化已实现（OLS截面回归），但缺少市值中性化。A股的小盘溢价在2021-2025年间年均约12%，任何不加市值中性化的因子都有严重的规模偏差。
- **建议：** 
  - 从 Baostock 日K `close * volume` 或 Qlib `$market_cap` 获取市值数据
  - 扩展 `neutralize_factor()` 支持 `method="market_cap"` 和 `method="industry+market_cap"`
  - 在前端中性化选择器中增加 "行业+市值中性化" 选项
- **理由：** 升为严重是因为：做完分组收益分析后，如果发现Q1(最小市值)收益显著高于Q5(最大市值)，无法区分是因子alpha还是规模溢价。

### 🟡 中等（7项）

#### 5. 因子筛选/降维机制缺失
- **来源：** 第一轮 一-3
- **现状：** 因子相关性分析端点已存在（`/api/factors/correlation`），但没有系统性的筛选建议。158个因子中大量冗余——多个波动率变体、多个价格位置指标高度相关。
- **建议：** 基于 IC 相关性矩阵做层次聚类（阈值0.7），每簇保留ICIR最高的因子。前端展示"有效独立因子数"和聚类树。
- **理由：** 因子共线性 → 模型过拟合 → 样本外衰减。第一步的因子筛选比模型超参调整重要得多。

#### 6. 交易成本模型过于简化
- **来源：** 第一轮 二-2
- **现状：** 固定佣金率0.03% + 固定0.03%费率。对沪深300大盘股勉强适用，对中证500/1000不适用。
- **建议：** 至少加入简单的平方根冲击模型：`impact = 0.0003 + 0.1 * sigma_daily * sqrt(trade_size / daily_volume)`
- **理由：** 30只持仓、月度调仓的小盘策略，忽略可变冲击成本虚高收益2-5%/年。

#### 7. 组合优化缺少换手率约束
- **来源：** 第一轮 三-3
- **现状：** `_min_variance`、`_max_sharpe`、`_risk_parity` 三种优化都没有换手率惩罚项。
- **建议：** 在目标函数中加入 `lambda_turnover * sum(|w_i - w_i_prev|)`，或至少展示优化前后的换手率对比。
- **理由：** 单边换手50%在A股约等于1.5%的成本（佣金+冲击+印花税），一年调仓12次就是18%——直接吃掉大部分alpha。

#### 8. 相关性矩阵是静态的
- **来源：** 第一轮 四-2
- **现状：** `_compute_correlation_matrix()` 使用全样本 Pearson 相关。危机时相关性急剧上升（diversification breakdown）完全不可见。
- **建议：** 分"上涨月"和"下跌月"展示两套相关性矩阵，标注差异。高差异的配对是尾部风险来源。
- **理由：** "危机来了才发现分散化是假的"是组合管理最痛的教训。

#### 9. Alpha158 每请求重复计算
- **来源：** 第一轮 七-2
- **现状：** 每次 `/api/factors/analyze` 重新构建 DatasetH + 158因子 + 数据处理（60-90秒）。同一日期范围被反复请求。
- **建议：** 
  - 按日期范围做内存缓存（`{(start, end): dataset}`），TTL 30分钟
  - 或 application startup 时预计算最近2年的因子数据
- **理由：** 当前每次分析等1分钟以上的用户体验很差，且浪费计算资源。

#### 10. 配对交易使用一次性协整检验
- **来源：** 第一轮 六-2
- **现状：** 全样本协整。但A股同行业股票的协整关系在政策和市场结构变化后会永久漂移。
- **建议：** 加入滚动协整检验（rolling 60天窗口），展示协整系数和ADF p值的时变性。
- **理由：** 全样本协整的配对在样本外有50%以上概率失效（A股实证）。

#### 11. ETF 轮动信号质量薄弱
- **来源：** 第一轮 六-4
- **现状：** 信号基于简单动量排名，无风险调整、无趋势过滤、无相关性过滤。
- **建议：** 使用风险调整动量（动量/波动率），加入60日均线趋势过滤（线下不做多），加入ETF间相关性过滤（同质性ETF只选最强）。
- **理由：** ETF轮动是最容易直接产生交易行为的模块，信号质量直接影响用户收益。

### 🟢 轻微（4项）

#### 12. 多数据源一致性验证
- **来源：** 第一轮 七-1
- **建议：** 在数据健康检查中加入 Qlib vs Baostock vs akshare 对同一标的收盘价的交叉验证（差异 > 1% 告警）。

#### 13. 有效前沿实用性改进
- **来源：** 第一轮 三-4
- **建议：** 在已有权重约束的基础上，区分"理想前沿"（无约束MV边界）和"可行前沿"（加入单票上限/行业上限后）。

#### 14. Daily Checklist 动态化
- **来源：** 第一轮 四-4
- **建议：** 基于当日真实数据生成动态清单：波动率异常检测、停复牌事件、北向资金异常流入流出。

#### 15. 宏观指标缓存
- **来源：** 第一轮 五-3
- **建议：** Redis 或内存缓存 akshare 宏观指标，TTL 1小时。a股宏观7指标串行获取延迟约8-12秒。
